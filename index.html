Principle 1. Squash and Stretch.
Gives a sense of mass and catches the eye.

@keyframes squash-stretch {
  0% {
    transform: translate(0);
    height: 250px;
    width: 100px;
  }
  
  50% {
    height: 100px;
    width: 250px;
  }
  
  100% {
    transform: translate(100%);
    height: 250px;
    width: 100px;
  }
}

Principle 4. Straight Ahead & Pose to Pose.
Animation in the browser can be done in:
  • Transitions - Straight Ahead
  • Keyframes - Pose to Pose

// This is an example for Straight Ahead animation.
// Here, the browser decides the interpolation between two states.

transition: height width 0.5s linear;
						

// Here, there is an example for Pose to Pose animation.
// The animator/developer will decide the interpolation (poses) from animation start to end.
@keyframes grow {
	10% {
		transform: scale(0);
	}

	50% {
		transform: scale(0.3, 0.3);
	}

	100% {
		transform: scale(1, 1);
	}
}

Principle 6. Slow In & Slow Out.
Nothing but robots move linearly. Naturalise. Appeal.
Linear
Ease-in-out

// Always always always always always always avoid
transition: all 1s linear;

// A good rule of thumb would be to always ease the movement or use a cubic bezier function if you feel fancy
transition: all 1s ease-in-out;
transition: all 1s cubic-bezier(0.5, 0.1, 1, 0.20);

Avoid linear at all costs if you don't want your graphic to look like an old Powerpoint.

Principle 7. Arcs.
Try to avoid straight lines. Real life motion is never done in such fashion.

Principle 9. Timing.
Motion is proportional to object mass. More frames = action is slower. Snappy.

UI actions are usually fast.

Fast information animations confuse the user.

// Good for UI

Principle 10. Exaggeration.
Relatable. Too realistic is unappealing. Draw attention.

// Exaggerate scale or color for special emphasis
// Remember to use ease principle!
// A good rule of thumb would be to overscale or underscale elements :)
@keyframes exaggeration {
  0% {
    transform: scale(1,1);
  }
  
  30% {
    transform: scale(1.3, 1.3);
  }
  
  100% {
    transform: scale(1, 1);
  }
}

Do's and dont's

  • Ask a colleague
  • Make animations long
  • Keep it consistent
  • Subtle is usually the best way to go
  • Preview. Preview. Preview.
  • Consider different devices
  • EXPERIMENT A LOT!
  • Over-do animations
  • Saturate the layout with animations
  • Animate everything
  • Force animation (ie. motion sickness)
  • Forget your audience
